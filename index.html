Raycast Car Controller
Raycast suspension from scratch - no wheel colliders. Surface behaviour data-driven via ScriptableObjects; add a road type without touching C#.
Raycast suspension - Hooke's law spring + damping per wheel, force applied at contact point
Grip circle - lateral and longitudinal forces treated as a 2D vector; combined magnitude capped by normal force × grip coefficient
Weight transfer - derived from CoG height, wheelbase, and trackwidth; longitudinal + lateral components smoothed toward target each FixedUpdate
Automatic gears - RPM-based shift-up/shift-down with hysteresis band to prevent gear hunting
SurfaceType ScriptableObjects - grip curve, rolling resistance, brake/accel multipliers, suspension damping, particle effects, all tunable per surface asset
Per-wheel particles - spawn/swap/stop drift particles when surface changes mid-slide; lifetime-aware cleanup
DOTween parking sequence - cinematic slow-down → rotation → reverse-into-spot animation, triggered by selection event
rayCastCar.cs - weight transferC#
// Weight transfer = (acceleration × CoG height × mass) / axle distance
// Derived from vehicle dynamics first principles - not a lerp approximation.
// Result feeds into each wheel's normalForce, which drives the grip circle.
private void CalculateLoadTransfer()
{
    Vector3 localAccel = transform.InverseTransformDirection(
        (carRigidbody.linearVelocity - previousVelocity) / Time.fixedDeltaTime);

    float totalMass = carRigidbody.mass;

    float longitudinal = Mathf.Clamp(
        (localAccel.z * centerOfGravityHeight * totalMass) / wheelbase,
        -totalMass * maxLoadTransferAmount, totalMass * maxLoadTransferAmount);

    float lateral = Mathf.Clamp(
        (localAccel.x * centerOfGravityHeight * totalMass) / trackWidth,
        -totalMass * maxLoadTransferAmount, totalMass * maxLoadTransferAmount);

    float front = totalMass * frontToRearWeightRatio       - longitudinal;
    float rear  = totalMass * (1f - frontToRearWeightRatio) + longitudinal;

    targetWheelLoads[0] = rear  / 2f - lateral; // Rear Left
    targetWheelLoads[1] = front / 2f - lateral; // Front Left
    targetWheelLoads[2] = rear  / 2f + lateral; // Rear Right
    targetWheelLoads[3] = front / 2f + lateral; // Front Right

    // Smooth toward target - prevents instantaneous weight snap
    for (int i = 0; i < 4; i++)
    {
        currentWheelLoads[i] = Mathf.MoveTowards(currentWheelLoads[i], targetWheelLoads[i],
            transferRate * totalMass * Time.fixedDeltaTime);
        currentWheelLoads[i] = Mathf.Clamp(currentWheelLoads[i],
            totalMass * 0.05f, totalMass * 0.8f);
        wheels[i].dynamicWeight = currentWheelLoads[i];
    }

    previousVelocity = carRigidbody.linearVelocity;
}
rayCastCar.cs - surface data per wheelC#
// Each wheel raycasts the ground and reads a SurfaceType ScriptableObject from it.
// Different surfaces mean different suspension feel, grip, and particles - per wheel,
// so the front can be on asphalt while the rear is on gravel.
private void CalculateSuspension(WheelData wheel)
{
    float offset = wheel.suspensionRestDistance - wheel.hit.distance;
    float vel    = Vector3.Dot(wheel.wheelTransform.up,
                       carRigidbody.GetPointVelocity(wheel.wheelTransform.position));

    // suspensionDampingMultiplier from SurfaceType asset (gravel, asphalt, mud…)
    float force = (offset * wheel.springStrength)
                - (vel   * wheel.springDamper * wheel.currentSurface.suspensionDampingMultiplier);

    // Dynamic weight from load transfer feeds directly into normal force
    float weightForce = wheel.dynamicWeight * 9.81f;
    wheel.normalForce  = Mathf.Max(0, force + weightForce);

    carRigidbody.AddForceAtPosition(
        wheel.wheelTransform.up * (force + weightForce),
        wheel.wheelTransform.position);
}

cameraEffects.cs - speed-reactive cameraC#
// All effects share one normalised speed value t (0 = stopped, 1 = max speed).
// FOV, orbital radius, tilt, shake, and post-processing all read from it -
// tunable independently, but guaranteed to stay in sync.
void LateUpdate()
{
    float t = Mathf.Clamp01(targetRigidbody.linearVelocity.magnitude * 3.6f / maxSpeedKmh);

    cam.Lens.FieldOfView = Mathf.Lerp(cam.Lens.FieldOfView, Mathf.Lerp(baseFOV,    maxFOV,    t), Time.deltaTime * fovSmooth);
    orbital.Radius       = Mathf.Lerp(orbital.Radius,       Mathf.Lerp(baseRadius, maxRadius, t), Time.deltaTime * radiusSmooth);
    cam.Lens.Dutch       = Mathf.Lerp(cam.Lens.Dutch,       maxTilt * t,                          Time.deltaTime * tiltSmooth);

    // Perlin noise shake - kicks in above threshold, scales with excess speed
    if (enableShake && t > shakeSpeedThreshold)
    {
        float shakeT = (t - shakeSpeedThreshold) / (1f - shakeSpeedThreshold);
        shakeTimer  += Time.deltaTime * shakeFrequency;

        orbital.TargetOffset = baseTargetOffset + new Vector3(
            Mathf.PerlinNoise(shakeTimer, 0f)         - 0.5f,
            Mathf.PerlinNoise(0f, shakeTimer)         - 0.5f,
            Mathf.PerlinNoise(shakeTimer, shakeTimer) - 0.5f
        ) * shakeIntensity * shakeT;
    }
    else { orbital.TargetOffset = baseTargetOffset; }

    // Post-processing: motion blur, film grain, lens distortion, bloom, DoF -
    // all driven by the same t, each with its own smooth speed and intensity range.
    if (enableMotionBlur && motionBlur != null)
        motionBlur.intensity.value = Mathf.Lerp(motionBlur.intensity.value,
            Mathf.Lerp(baseMotionBlurIntensity, maxMotionBlurIntensity, t),
            Time.deltaTime * motionBlurSmooth);
}
Stencil Buffer Shader
Objects only visible inside the headlight cones. Cone mesh renders first - writes a stencil ID, draws nothing. World objects skip pixels where the ID doesn't match.
Queue Geometry-1 - cone mesh renders before everything else in the frame
Blend Zero One + ZWrite Off - draws nothing visually; acts as a pure GPU mask
Stencil Pass Replace - stamps the ID wherever the cone covers, regardless of what was there
World objects use Stencil Comp Equal - only draw where the ID matches; invisible everywhere else
stencilBuffer.shader - headlight maskHLSL
// Applied to the headlight cone mesh. Renders nothing visible -
// only writes a stencil ID to every pixel the cone covers.
// World objects with Stencil { Comp Equal } skip drawing outside it.
Shader "Custom/StencilMask"
{
    Properties
    {
        [IntRange] _StencilID ("Stencil ID", Range(0, 255)) = 0
    }
    SubShader
    {
        Tags { "Queue"="Geometry-1" } // render before everything else

        Pass
        {
            Blend Zero One  // discard own colour - purely a mask
            ZWrite Off

            Stencil
            {
                Ref [_StencilID]
                Comp Always
                Pass Replace   // stamp ID wherever the cone mesh covers
            }
        }
    }
}

InkDialogueVariables.cs - persistent story stateC#
// Ink's Story resets on each run. This class captures all global variables
// at startup and syncs them back in at the start of every session,
// so the story remembers what happened in previous conversations.
public class InkDialogueVariables
{
    private Dictionary<string, Ink.Runtime.Object> variables;

    public InkDialogueVariables(Story story)
    {
        variables = new Dictionary<string, Ink.Runtime.Object>();
        foreach (string name in story.variablesState)
            variables.Add(name, story.variablesState.GetVariableWithName(name));
    }

    public void SyncVariablesAndStartListening(Story story)
    {
        SyncVariablesToStory(story);
        story.variablesState.variableChangedEvent += UpdateVariableState;
    }

    public void StopListening(Story story) =>
        story.variablesState.variableChangedEvent -= UpdateVariableState;

    public void UpdateVariableState(string name, Ink.Runtime.Object value)
    {
        if (variables.ContainsKey(name)) variables[name] = value;
    }

    private void SyncVariablesToStory(Story story)
    {
        foreach (KeyValuePair<string, Ink.Runtime.Object> variable in variables)
            story.variablesState.SetGlobal(variable.Key, variable.Value);
    }
}
Event Bus & Input State Machine
Central event bus - no direct references between systems. Input state machine keeps one action map active at a time; no conflicting inputs.
gameEventsManager singleton - holds typed event channel objects (DialogueEvents, InputEvents, SelectionEvents); all wiring uses C# events, not Unity message passing

// Single scene-wide singleton that owns all typed event channels.
// Systems subscribe to whichever channels they care about -
// no direct references between them, no FindObjectOfType calls.
public class gameEventsManager : MonoBehaviour
{
    public static gameEventsManager instance { get; private set; }

    public DialogueEvents  dialogueEvents;
    public InputEvents     inputEvents;
    public SelectionEvents selectionEvents;
    public CarInputAction  inputActions;

    private void Awake()
    {
        if (instance != null) { Debug.LogError("Duplicate GameEventsManager!"); return; }
        instance        = this;
        inputActions    = new CarInputAction();
        dialogueEvents  = new DialogueEvents();
        inputEvents     = new InputEvents();
        selectionEvents = new SelectionEvents();
    }
}
